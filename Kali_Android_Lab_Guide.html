⚙️ LAB SETUP: Prerequisites & Installation

What You Need:

  • ✅ Kali Linux (VM or native)
  • ✅ Android phone (rooted recommended, non-rooted works too)
  • ✅ USB cable (USB debugging enabled)
  • ✅ ~5GB disk space

Step 1: Enable ADB on Android Phone

PHONE SETUP:
  1. Settings → About phone → Build number
  2. Tap "Build number" 7 times (until "Developer mode enabled")
  3. Back to Settings → Developer options
  4. Enable "USB debugging"
  5. Enable "Wireless debugging" (optional, for network testing)
  6. Connect phone to Kali with USB cable

Step 2: Install Kali Requirements

$ Install Android SDK Tools on Kali
# Update Kali sudo apt-get update && sudo apt-get upgrade -y # Install ADB and Fastboot sudo apt-get install -y android-sdk-platform-tools android-sdk-build-tools # Install Frida pip install frida frida-tools # Install other essential tools sudo apt-get install -y \ openjdk-11-jdk \ python3-dev \ build-essential \ git \ unzip \ wget # Verify installations adb version frida --version
✓ Output should show:
Android Debug Bridge version 1.0.41 frida 16.0.8

Step 3: Connect Phone to Kali

$ Connect and authorize phone
# List connected devices adb devices # First time, phone will ask permission - TAP ALLOW # Verify connection adb devices -l # Get device info adb shell getprop ro.build.version.release # Restart ADB as root (for some operations) adb root adb shell

🎣 FRIDA: Runtime Function Hooking (NO APK RECOMPILATION!)

Frida lets you intercept and modify app behavior at runtime without touching the APK. This is the professional way to test security.

Step 1: Install Frida Server on Phone

$ Install Frida server
# Check your phone architecture adb shell getprop ro.product.cpu.abi # Download appropriate Frida server # For ARM64 (most modern phones): wget https://github.com/frida/frida/releases/download/16.0.8/frida-server-16.0.8-android-arm64.xz # For ARM (older phones): # wget https://github.com/frida/frida/releases/download/16.0.8/frida-server-16.0.8-android-arm.xz # Extract unxz frida-server-16.0.8-android-arm64.xz # Push to phone adb push frida-server-16.0.8-android-arm64 /data/local/tmp/frida-server # Make executable adb shell chmod +x /data/local/tmp/frida-server # Run Frida server (in background) adb shell "/data/local/tmp/frida-server &" # Verify it's running adb shell ps | grep frida
✓ You should see frida-server in process list

Step 2: List Running Apps

$ List all running apps
# From Kali, list all apps on phone frida-ps -U # Or list specific app frida-ps -U | grep instagram # Get package name adb shell pm list packages | grep -i app_name
✓ Output shows:

Step 3: Write Your First Frida Hook

hook_example.js - Simple function interception
// Hook Activity lifecycle Java.perform(function() { var Activity = Java.use("android.app.Activity"); Activity.onCreate.implementation = function(savedInstanceState) { console.log("[*] Activity.onCreate called!"); return this.onCreate(savedInstanceState); }; Activity.onResume.implementation = function() { console.log("[*] Activity.onResume called!"); return this.onResume(); }; }); // Hook SharedPreferences (read stored data) Java.perform(function() { var SharedPreferences = Java.use("android.content.SharedPreferences"); SharedPreferences.getString.implementation = function(key, defValue) { var result = this.getString(key, defValue); console.log("[+] SharedPref read - Key: " + key + " Value: " + result); return result; }; }); // Hook MessageDigest (intercept hashing) Java.perform(function() { var MessageDigest = Java.use("java.security.MessageDigest"); MessageDigest.getInstance.implementation = function(algorithm) { console.log("[!] Hashing algorithm used: " + algorithm); return this.getInstance(algorithm); }; });

Step 4: Launch Frida Hook

$ Attach Frida to running app
# Option 1: Hook running app frida -U -p PID -l hook_example.js # Option 2: Launch app and hook it frida -U -f com.target.app -l hook_example.js --no-pause # Option 3: Spawn app with hook frida -U -f com.whatsapp -l hook_example.js # -U = Use device (USB) # -f = Spawn and attach to app # -l = Load JavaScript hook # --no-pause = Don't pause on startup
✓ Output shows hook working:

Technique 1: Bypass Certificate Pinning

bypass_certificate_pinning.js
// Bypass SSL/TLS certificate pinning Java.perform(function() { var X509TrustManager = Java.use("javax.net.ssl.X509TrustManager"); var TrustManager = Java.registerClass({ name: "TrustManager", implements: [X509TrustManager], methods: { checkClientTrusted: function(certs, authType) {}, checkServerTrusted: function(certs, authType) {}, getAcceptedIssuers: function() { return []; } } }); var sslContext = Java.use("javax.net.ssl.SSLContext"); sslContext.init.overload( "[Ljavax/net/ssl/KeyManager;", "[Ljavax/net/ssl/TrustManager;", "Ljava/security/SecureRandom;" ).implementation = function(keyManagers, trustManagers, secureRandom) { console.log("[!] Certificate pinning bypassed!"); return this.init(keyManagers, [new TrustManager()], secureRandom); }; }); // Alternative: Hook OkHttp certificate pinning Java.perform(function() { var CertificatePinner = Java.use("okhttp3.CertificatePinner"); CertificatePinner.check.overload("java.lang.String", "java.util.List").implementation = function(hostname, list) { console.log("[+] Certificate pinning check for: " + hostname); // Don't verify, just return return; }; });

📦 MANUAL APK MODIFICATION (Extract, Modify, Rebuild)

Overview

When you need to permanently modify an app (not just runtime), extract it, modify files, repack, sign, align, and install.

Step 1: Extract APK

$ Extract APK to work directory
# Get APK from phone adb shell pm path com.target.app # Example output: # package:/data/app/com.target.app-abc123/base.apk adb pull /data/app/com.target.app-abc123/base.apk ./target.apk # Or pull from any installed app: adb pull $(adb shell pm path com.target.app | cut -d : -f 2) target.apk # Create work directory mkdir apk_work cd apk_work # Extract APK (it's just a ZIP file!) unzip -q ../target.apk # List contents ls -la # You can now edit: # - res/values/strings.xml (text resources) # - assets/* (any static files) # - AndroidManifest.xml (app configuration) # - smali/* (bytecode - advanced users)
✓ Contents of APK:
AndroidManifest.xml classes.dex resources.arsc res/ assets/ lib/ META-INF/

Step 2: Modify Files (Examples)

$ Modify app strings
cd apk_work # Edit string resources nano res/values/strings.xml # Change app name sed -i 's/Original App Name/Modified App/g' res/values/strings.xml # Change version string sed -i 's/Version 1.0/Version 1.0 (Modified)/g' res/values/strings.xml # Add debug message echo 'TRUE' >> res/values/strings.xml # Modify AndroidManifest to disable certificate pinning sed -i 's/android:usesCleartextTraffic="false"/android:usesCleartextTraffic="true"/g' AndroidManifest.xml # Or create cleartext config cat > res/xml/network_security_config.xml << 'EOF' example.com EOF

Step 3: Repack APK

$ Repack modified files into APK
cd apk_work # Create new APK from modified files zip -r ../app-modified.apk * -q cd .. # Verify new APK ls -lh app-modified.apk # Check contents unzip -l app-modified.apk | head -20

Step 4: Sign APK

$ Create keystore and sign
# Create keystore (first time only) keytool -genkey -v -keystore ~/.keystore \ -keyalg RSA -keysize 2048 -validity 10000 \ -alias my_key \ -storepass password \ -keypass password \ -dname "CN=MyApp, O=MyOrg, C=US" # Sign the APK jarsigner -verbose -sigalg SHA256withRSA -digestalg SHA256 \ -keystore ~/.keystore \ -storepass password \ -keypass password \ app-modified.apk my_key # Verify signature jarsigner -verify -verbose -certs app-modified.apk

Step 5: Align APK (CRITICAL!)

$ Align APK for optimal performance
# This step is ESSENTIAL - fixes most installation errors! zipalign -v 4 app-modified.apk app-final.apk # Verify alignment zipalign -c -v 4 app-final.apk # Verify signature again after alignment jarsigner -verify -verbose -certs app-final.apk
✓ Alignment verification output:
4-byte alignment: OK Signature: OK

Step 6: Install on Phone

$ Uninstall old version and install modified
# Uninstall original app adb uninstall com.target.app # Install modified APK adb install -r app-final.apk # Verify installation adb shell pm list packages | grep target.app # Launch app adb shell am start -n com.target.app/.MainActivity # View app logs adb logcat | grep "target.app"

#!/bin/bash # Colors for output RED='\033[0;31m' GREEN='\033[0;32m' YELLOW='\033[1;33m' BLUE='\033[0;34m' NC='\033[0m' # No Color echo -e "${BLUE}═══════════════════════════════════════${NC}" echo -e "${BLUE} KALI + ANDROID LAB AUTOMATION${NC}" echo -e "${BLUE}═══════════════════════════════════════${NC}" # Check prerequisites echo -e "${YELLOW}[*] Checking prerequisites...${NC}" if ! command -v adb &> /dev/null; then echo -e "${RED}[!] ADB not found. List connected devices" echo "2. Install Frida server" echo "3. List running apps" echo "4. Extract APK (by package name)" echo "5. Repack modified APK" echo "6. Sign APK" echo "7. Align APK" echo "8. Full workflow (extract→sign→align→install)" echo "9. Run Frida hook" echo "0. Exit" read -p "Choice: " choice case $choice in 1) list_devices ;; 2) install_frida_server ;; 3) frida-ps -U ;; 4) read -p "Package name: " pkg extract_apk "$pkg" ;; 5) read -p "Package name: " pkg repack_apk "$pkg" ;; 6) read -p "APK file: " apk sign_apk "$apk" ;; 7) read -p "APK file: " apk align_apk "$apk" ;; 8) read -p "Package name: " pkg extract_apk "$pkg" repack_apk "$pkg" sign_apk "$pkg-modified.apk" align_apk "$pkg-modified.apk" install_apk "$pkg-modified.apk" ;; 9) read -p "Hook file: " hook read -p "Package name: " pkg frida -U -f "$pkg" -l "$hook" --no-pause ;; 0) exit 0 ;; *) echo -e "${RED}Invalid choice${NC}" ;; esac done
